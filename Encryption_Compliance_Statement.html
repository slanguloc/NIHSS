Encryption Compliance Statement

App: Zysquy (NIH Stroke Scale for non-English speaking patients)

This document describes the use of encryption in the above-named app for App Store export compliance.

1. Use of encryption

The app uses encryption only as follows:

  • Local data storage: Data stored on the device (e.g., user language preference and assessment history) is encrypted using Apple’s CryptoKit framework (AES-GCM). This is encryption provided within Apple’s operating system; the app does not implement any proprietary or non-standard algorithms.
  • Network communication: Any network communication uses standard HTTPS/TLS as provided by the operating system.

2. Proprietary or non-standard algorithms

The app does not contain or use any encryption algorithms that are proprietary or that are not accepted as standard by international standards bodies (e.g., IEEE, IETF, ITU). All encryption is performed via Apple’s frameworks (CryptoKit, URLSession/HTTPS).

3. Standard encryption in addition to Apple’s OS

The app does not use standard encryption algorithms instead of, or in addition to, the encryption within Apple’s operating system in a manner that would require export documentation. The app uses only encryption that is accessed through Apple’s operating system (CryptoKit and system TLS).

4. Conclusion

This use of encryption is exempt from the requirement to submit formal encryption registration (ERN) or classification documentation (CCATS). No separate encryption documentation is required beyond this statement.

Submitted for App Store Connect export compliance.
